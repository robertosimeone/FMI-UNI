CSS Object Model - CSSOM

Какво е CSS Обектният Модел? Как работи?

Какво е CSS Обектният модел?

"CSS обектният модел е набор от API, позволяващи манипулирането на CSS от JavaScript. Много прилича на DOM, но e за CSS, а не за HTML. Той позволява на потребителите да четат и променят динамично CSS стила."

- MDN

Как работи?

За разлика от Document Object Model (DOM), който разглежда HTML документа на страницата, CSS Object Model работи заедно с DOM и се занимава с изобразяването на документа със стиловата таблица. Стиловата таблица е свързана и анализирана към браузъра чрез HTML, където CSS обектният модел се комбинира с DOM, за да образува дърво за изобразяване. Това дърво за изобразяване съдържа необходимите възли за изобразяване на страницата.

Процесът на рендиране на CSS обектния модел и DOM се извършва в рамките на браузъра и се нарича критичен път на рендиране. Това е процес или поредица от стъпки, през които преминава браузърът, за да преобразува HTML, CSS и JavaScript в пиксели, които се показват на екрана. Този път включва CSS обектен модел, DOM, оформление и дърво за изобразяване, както е показано на фигурата по-долу. Ето стъпките в критичния път на рендиране:

  1. Разработка на DOM, CSS обектен модел и изпълнение на JavaScript.
  2. Браузърът зарежда HTML файла.
  3. Заявка към JavaScript и CSS файловете съответно чрез < script > и < link > и тагове.
  4. JavaScript чрез DOM прави необходимите промени и актуализира документа.
  5. Layout изчислява точната позиция и размер на всеки обект.
  6. След това CSS се анализира като компилация на CSS обектен модел и променя стила в DOM.
  7. Преобразуване в реални пиксели и показването им на екрана

Енджинът на браузъра превежда този процес и създава дървото за изобразяване. Веднага след като оформлението и начина, по който всеки елемент трябва да бъде позициониран, са определени, пикселите се рисуват от браузъра на екрана. Също така е важно да се има предвид, че независимо дали добавяме или не стилови правила към уеб страницата, браузърът по подразбиране присвоява вградени стилове на документа, зареден в браузъра; това се нарича user-agent или user-agent-stylesheet. Въпреки това разработчиците на интерфейса или уеб дизайнерите създават своя външен стилов лист, наречен авторски стилов лист; този CSS стил е свързан с HTML файла. Това е най-често срещаният стилов лист, тъй като разработчиците ги пишат, за да коригират и актуализират стиловия лист на потребителския агент.

Сега нека разгледаме различните интерейси и функционалностите,които те предлагат.

CSSStyleDeclaration интерфейс

Интерфейсът CSSStyleDeclaration представлява обект, който е блок за CSS декларация, и излага информация за стил и различни методи и свойства, свързани със стила.

Има три начина, по които може да бъде достъпен един обект от тип CSSStyleDeclaration

  • HTMLElement.style

    Това е най-основния начин,по който може да манипулираме или достъпим CSS property-та чрез JavaScript.HTMLElement.style се грижи за inline стиловете на един елемент. 

                                    
document.body.style.backgroundColor = "green";

    На горния пример сменяме css property-то background-color на ‘green’. За изписването на css property-тата в javascript се използва camelCase конвенцията, както можем да видим в по-горния пример(background-color -> backgroundColor).Изкючение от това правило прави единствено float,понеже float e запазена дума в javascript , css property-to float e заменено с cssFloat.

const f = document.getElementById("ball");
const div = document.getElementById("ccc");
function getPosition(el) {
    var xPosition = 0;
    var yPosition = 0;

    while (el) {
    if (el.tagName == "BODY") {
        // deal with browser quirks with body/window/document and page scroll
        var xScrollPos = el.scrollLeft || document.documentElement.scrollLeft;
        var yScrollPos = el.scrollTop || document.documentElement.scrollTop;

        xPosition += (el.offsetLeft - xScrollPos + el.clientLeft);
        yPosition += (el.offsetTop - yScrollPos + el.clientTop);
    } else {
        xPosition += (el.offsetLeft - el.scrollLeft + el.clientLeft);
        yPosition += (el.offsetTop - el.scrollTop + el.clientTop);
    }

    el = el.offsetParent;
    }
    return {
    x: xPosition,
    y: yPosition
    };
}
div.addEventListener("click",getClickPosition,false);
function getClickPosition(e){
    var parentPosition = getPosition(div);
    var xPosition = e.clientX - parentPosition.x - (f.offsetWidth/2);
    var yPosition = e.clientY -parentPosition.y -(f.offsetHeight/2);
    var translate3dValue = "translate3d(" + xPosition + "px," +yPosition + "px,0)";
    f.style.transform = translate3dValue;
}

CSSStyleSheet

Интерфейсът CSSStyleSheet представлява CSS stylesheet,който позволява да се инспектира и модифицира списъка с правила, съдържащи се в stylesheet-а. Той наследява свойства и методи от своя родител - StyleSheet. Един stylesheet представлява колекция от CSSRule обекти,които представлявът съответно правило.CSSRulе обектите се съдържат в CSSRuleList,който от своя страна може да бъде достъпен чрез cssRules property-то.

Един CSSStyleSheet обект също така може да бъде конструиран чрез конструкторът CSSStyleSheet() и методите - CSSStyleSheet.replace() и CSSStyleSheet.replaceSync()(И двата метода заменят всички правила в определен stylesheet с тези,посочени в методите. Разликата между двата метода е,че единия го прави синхронно,докато другия асинхронно.

Instance Properties

  • CSSStyleSheet.cssRules(read only) - връща CSSRuleList обект,който съдържа всички CSSRu le обекти от съответния stylesheet
  • CSSStyleSheet.ownerRule(read only) - ако stylesheet-a е импортнат посредством @import правило,то се връща съответното CSSImportRule,null иначе.

Instance Methods

  • CSSStyleSheet.deleteRule() - трие правило на подадения индекс.
  • CSSStyleSheet.insertRule() - въвежда ново правило на определен индекс.Ако индекс не е подаден по default се добавя на индекс 0.
  • CSSStyleSheet.replace(),CSSStyleSheet.replaceSync() - заменят съдържанието на stylesheet с друго,подадено като string(.replace връща Promise обект)

Legacy Properties

Въведени от Microsoft,като се поддържат за съвместимост със съществуващи сайтове

  • CSSStyleSheet.rules(read only) - връща обект,съдържащ всички правила на определен stylesheet(сходно на cssRules)
  • CSSStyleSheet.addRule(selector,styleBlock,index=length) - Въвежда правило по подаден selector и styleBlock.Различава се от метода insertRule(),по това че в insertRule() цялото правило се подава като един string
  • CSSStyleSheet.removeRule(index) - идентично на deleteRule().Премахва правило от даден индекс.

Как да достъпим един styleSheet обект?

CSSStyleSheet обект за даден документ може да бъде достъпен чрез Document.styleSheets.Един StyleSheet обект автоматично се създава и добавя към Document.styleSheets от браузърът, когато един styleSheet е зареден за конкретен документ.

MediaList

Интерфейсът MediaList представлява медийните заявки на stylesheet

Instance Properties

  • MediaList.mediaText - връща низ,който репрезентира MediaList интерфейса.(media заявките на даден stylesheet,всяка разделена с ",",например screen and (min-width: 480px), print.Това property също така може да се използва за да зададем нови медийни заявки за документа,като те трябва да са раздлеени с запетайки,например "screen,print".Ако на mediaText му бъде зададена стойност null,ще бъде третиран като празен низ("")
  • MediaList.length - връща броя на media заявките в MediaList обекта.

Instance Methods

  • MediaList.appendMedium() - Методът приема media заявка.(например "print".)добавя нова media заявка към съответния MediaList обект.Ако media заявката вече е в MediList обекта,то повторното добавяне се игнорира.
  • MediaList.deleteMedium() - Премахва media заявка по подадено име("print").Ако такава не съществува се хвърля Exception
  • MediaList.item() - връща конкретнa media заявка по подадене индекс като параметър.Сходно на [](Разликата е в това че ако подадения index е out of bounds при .item() се връща null,а при [] - undefined).

CSSKeyframeRule

Интерфейсът CSSKeyframeRule описва обект, представляващ набор от стилове за keyframe. Той съответства на съдържанието на единичен keyframe от @keyframes at-rule

Instance Properties

  • CSSKeyframeRule.cssText - Представлява ключа на ключовия кадър, като „10%“, „75%“. Ключовата дума from съответства на „0%“, а ключовата дума to на „100%“
  • CSSKeyframeRule.style – връща CSSStyleDeclaration обект, асоцииран с съответния keyframe.

Example

CSSKeyframesRule

Интерфейсът CSSKeyframesRule описва обект, представляващ пълен набор от ключови кадри за CSS анимация. То съответства на съдържанието на цяло @keyframes at-rule.

Instance Properties

  • CSSKeyframesRule.name - Представлява името на keyframes, използвани от property-то animation-name.
  • CSSKeyframesRule.cssRule – връща CSSRuleList обект ,съдържащ правилата в @keyframes правилото

Instance Methods

  • CSSKeyframesRule.appendRule() – въвежда ново keyframe правило в текущото @keyframes правило. Като параметър се дава едно keyframe правило,ако се подадат 2 се хвърля DOM Exception(Syntax ERR).
  • CSSKeyframesRule.deleteRule() – трие keyframe правило от текущото @keyframes праввило.На функцията се подава един параметър – ключовата дума(from,to) или процентна стойност(0%-100%).
  • CSSKeyframesRule.findRule() – приема един параметър – ключовата дума(from,to) или процентна стойност(0%-100%).Връща последното намерено правило, което кореспондира на подадения параметър.Ако такова правило не е намерено не се връща нищо.
